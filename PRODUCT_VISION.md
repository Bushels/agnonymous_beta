# Agnonymous Product Vision

Last updated: 2026-04-24

## Current Decision

Agnonymous is a mobile-first anonymous agriculture posting board.

The relaunch is a return to the original purpose: people can read, post, comment, vote, watch, and refresh without creating an account. Monette is the first-class launch room.

## Product Promise

Anonymous agriculture discussion. No sign up. No pricing board. Just the thread.

## Hard Boundaries

These are out of v1:

- Sign up or sign in.
- Username-first posting.
- Market intelligence dashboards.
- Fertilizer or chemical input pricing.
- Marketplace pricing.
- Cash bid tracking.
- My Farm, My Bins, breakeven calculators, or elevator maps.
- Gamification-first reputation loops.
- Funny reactions.

Older repo documents still describe some of those ideas. Treat them as legacy context unless this file is explicitly updated.

## Core V1 Loop

1. Reader opens the board on mobile.
2. Monette Room loads first.
3. Reader can switch rooms or search.
4. Reader taps the bottom icon-only post button.
5. Reader posts anonymously with category, title, details, and optional region.
6. Other readers comment anonymously.
7. Readers vote with thumbs up, neutral, or thumbs down.
8. Readers watch threads on their device and see new-comment markers.
9. Readers refresh repeatedly because the board feels live.

## User Identity Model

There is no account identity in v1.

The app uses a locally stored anonymous device ID:

- It is generated by `AnonymousIdService`.
- It is stored in local device/browser preferences.
- It is used for posts, comments, votes, and watched threads.
- It must not be treated like a user account.
- It must not be joined to auth profiles.

If browser storage is cleared, the watch list and local identity reset. That is acceptable for v1.

## Current Database Intent

Posts:

- Anonymous insert allowed.
- `user_id` is null for board posts.
- `anonymous_user_id` is required.
- `is_anonymous` is true.

Comments:

- Anonymous insert allowed.
- `anonymous_user_id` is required.
- Comment count backfills onto posts.

Votes:

- Allowed values remain `thumbs_up`, `partial`, `thumbs_down`, and legacy `funny`.
- UI exposes only `thumbs_up`, `partial`, and `thumbs_down`.
- `partial` means neutral.
- Existing `funny` rows are preserved but no longer exposed.

Watched threads:

- Stored locally immediately.
- Mirrored to `anonymous_post_watches` when the migration/RPCs are available.
- Used for new-comment counts.

## Notification Direction

In-app new-comment markers are v1.

Push notifications are a later opt-in step and do not require sign-in, but they do require:

- Notification permission prompt.
- FCM token capture.
- Anonymous device token table.
- Supabase Edge Function or backend sender.
- Generic privacy-safe payload such as "New comment on your watched thread."

Do not bend the existing auth-user notification table into anonymous notifications.

## Definition Of Done For Relaunch V1

- A new visitor can use the board without sign-in.
- Monette Room opens cleanly on mobile.
- Posting works anonymously.
- Commenting works anonymously.
- Voting works anonymously with three visible choices.
- Watch/new-comment state works on the same device.
- No pricing/dashboard/sign-in surfaces are visible in the main board flow.
- Loading screen and icon feel like the current app, not the old dashboard app.
- The docs point future agents to this direction first.
